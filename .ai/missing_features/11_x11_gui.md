## Missing (Critical)

### Scalable Font Rendering (FreeType-like)
**What**: The current 8√ó16 bitmap font can only display at one size and has limited character coverage.
Real applications (Firefox, GTK apps, Qt apps) require:
- TrueType/OpenType font loading
- Glyph rasterization at arbitrary sizes
- Subpixel rendering (ClearType-equivalent)
- Font metrics (ascent, descent, advance width per glyph)
- Unicode character coverage

**Current state**: `gdi/text.rs` renders glyphs from a hardcoded 8√ó16 bitmap array. No font files.

**Note**: A full FreeType port (~35,000 LOC) is out of scope. Options:
1. Embed a minimal TrueType rasterizer (stb_truetype.h = 3,700 LOC, port to Rust)
2. Use pre-rasterized bitmap font atlas for common sizes (8, 10, 12, 14pt)
3. Expose FreeType as a userspace library called from X11 clients

**Reference**: FreeType2 `src/truetype/ttinterp.c`; `stb_truetype.h` (single-file rasterizer)

---

### Clipboard / Selection (ICCCM)
**What**: X11 clipboard is implemented via property-based selection protocol (PRIMARY, CLIPBOARD,
SECONDARY selections). The owner of a selection responds to `SelectionRequest` events by writing
data to the requestor's property.

**Current state**: Atoms for CLIPBOARD/PRIMARY exist, but `SetSelectionOwner` / `GetSelectionOwner`
/ `ConvertSelection` / `SelectionNotify` event delivery are not implemented.

**Why critical**: Copy-paste between applications is entirely broken without this.

**Reference**: ICCCM 2.0 ¬ß2; `xclipboard` source; `XSetSelectionOwner(3)` man page

---

### Drag-and-Drop (XDND Protocol)
**What**: XDND (X Drag-and-Drop) protocol: source window sets `XdndAware` property, drags
send `XdndEnter`/`XdndPosition`/`XdndDrop` via `ClientMessage` events, target responds with
`XdndFinished`. Property-based data transfer.

**Why critical**: File manager ‚Üí app drops, URL bar drags, text drag in browsers.

**Reference**: XDND specification; freedesktop.org XDND protocol v5

---

### `SelectionRequest` / `SelectionNotify` Event Delivery
**What**: When a client calls `XConvertSelection()`, the X server sends a `SelectionRequest`
event to the selection owner. The owner writes the data and sends `SelectionNotify`.

**Current state**: `SelectionRequest` event type exists in `x11/event.rs` but
`ConvertSelection` request handler is missing/stub.

## Missing (High)

### XRender Full Compositing
**What**: Current RENDER implementation handles only `Src` and `Over` Porter-Duff operators.
Full RENDER protocol requires:
- All 13 standard operators (Clear, Src, Dst, Over, OverReverse, In, InReverse, Out, OutReverse,
  Atop, AtopReverse, Xor, Add, Saturate, DisjointSrc, etc.)
- Affine transform on pictures (`SetPictureTransform`)
- Scaling via `ChangePicture` filter (Nearest, Bilinear)
- Glyph rendering via `RenderAddGlyphs` / `RenderCompositeGlyphs*` ‚Äî core of text in Cairo/Pango

**Why high**: Cairo (used by Firefox, GTK, and everything that uses Pango) relies on
`RenderCompositeGlyphs` for all text rendering.

**Reference**: X Render extension protocol v0.11 spec; `xserver/render/render.c`

---

### Glyph Sets (`RenderCreateGlyphSet` / `RenderAddGlyphs` / `RenderCompositeGlyphs8/16/32`)
**What**: RENDER glyph operations let clients upload pre-rasterized glyphs (from FreeType)
into server-side glyph caches, then composite them at arbitrary positions. This is how
Cairo renders text ‚Äî not via XDrawString but via RENDER glyphs.

**Current state**: Not implemented.

**Reference**: X Render extension spec ¬ß4.6; `xserver/render/glyph.c`

---

### Window Manager Protocols (EWMH / ICCCM)
**What**: Extended Window Manager Hints (EWMH, freedesktop.org) define atoms and properties
that WMs and apps use to communicate:
- `_NET_WM_NAME` ‚Äî UTF-8 window title
- `_NET_WM_ICON` ‚Äî taskbar icon (ARGB32 array)
- `_NET_WM_STATE` ‚Äî fullscreen, maximized, minimized, modal
- `_NET_WM_WINDOW_TYPE` ‚Äî normal, dialog, tooltip, notification, splash
- `_NET_ACTIVE_WINDOW` ‚Äî which window has focus
- `_NET_WM_STRUT_PARTIAL` ‚Äî reserve space for panel/taskbar

**Current state**: Some atoms pre-seeded, but none of these protocols are honored by the WM.

**Reference**: EWMH spec (freedesktop.org); `_NET_SUPPORTED` root property

---

### Hardware Cursor Plane
**What**: Upload a custom cursor image to the GPU hardware cursor plane so that cursor movement
doesn't require redrawing the screen. Without hardware cursor, every mouse move triggers a
full compositor repaint.

**Current state**: VMware SVGA has hardware cursor registers, but the compositor likely
redraws the cursor in software.

**Reference**: VMware SVGA II programming guide (cursor FIFO commands); `linux/drivers/gpu/drm/vmwgfx/`

---

### Keyboard Map / XKB
**What**: XKB (X Keyboard Extension) manages keyboard layouts, modifier mappings, key symbols.
`XkbGetMap`, `XkbGetNames`, `XkbGetCompatMap`. Without a proper keymap, international keyboards
and modifier combos (Alt+key, Ctrl+Shift+key) are unreliable.

**Current state**: XKEYBOARD extension is registered (opcode 135) but handlers are stubs.

**Reference**: XKB protocol specification; `xkeyboard-config` key definition files

## GDI-Specific Gaps

The GDI layer in `gdi/` is used by the WM/compositor but has:
- **No arc/ellipse drawing**: `Ellipse()`, `Arc()`, `Chord()` not implemented
- **No bezier curves**: `PolyBezier()` not implemented
- **No gradient fills**: solid colors only
- **No alpha channel in DC**: all DCs are 24-bit RGB
- **No font quality settings**: always bitmap, no antialiasing
- **No pen styles beyond solid**: no dashed/dotted line in GDI

**Reference**: `XP/base/ntos/win32/ntgdi/` (gdi32 service table, ~200 GDI operations)

## Implementation Priority

1. **Clipboard/selection** (ICCCM ConvertSelection) ‚Äî copy-paste between apps
2. **RENDER glyph sets** ‚Äî RenderCreateGlyphSet + RenderAddGlyphs + RenderCompositeGlyphs8 ‚Äî enables Cairo text
3. **EWMH property handling** ‚Äî _NET_WM_STATE fullscreen/maximize; _NET_WM_NAME UTF-8 title
4. **XKB stub ‚Üí real keymap** ‚Äî proper modifier handling for international keyboards
5. **Gradient fills in GDI** ‚Äî needed for modern UI themes
6. **stb_truetype port** ‚Äî TrueType glyph rasterization for the server-side font cache
