# Code of Conduct

AstryxOS is a research-oriented open-source project. We want contributing to
feel collaborative, technically rigorous, and welcoming.

## The Standard

Participants in any AstryxOS space — issues, pull requests, merge requests,
code review, or real-time channels — are expected to:

- Treat other contributors with respect
- Give feedback on the technical merits of a change, not the person behind it
- Assume good faith when someone asks a naive question
- Credit prior work and cite upstream references accurately (ReactOS, Linux
  kernel, Rust community — giving credit is both correct and helpful)
- Keep the signal-to-noise ratio high: concrete repro steps beat opinions

Unacceptable behaviour includes:

- Personal attacks, insults, or derogatory comments about a contributor
- Harassment of any kind, public or private
- Deliberate misgendering or use of language a contributor has asked you
  to stop using
- Sexualised or otherwise sexually charged content in project spaces
- Deliberately disruptive behaviour: flamewars, spam, off-topic derailment
- Sharing another contributor's private information without consent
- Deliberately introducing proprietary source citations (Microsoft Windows
  source leaks, etc.) — this is both a legal and a social violation

## Enforcement

Maintainers are responsible for clarifying the standard and will take
proportionate action for violations, including:

1. A private note asking you to stop the behaviour
2. A public warning on the relevant thread
3. Temporary ban from project spaces
4. Permanent ban for repeated or severe violations

Maintainers may edit, delete, or reject contributions that violate this code.

## Reporting

`184648288+me1iissa@users.noreply.github.com` with the subject line
`[CoC] <short description>`. Reports are handled confidentially. We will
acknowledge within 7 days and share our proposed response within 14 days.

If the report concerns a maintainer, you may escalate through a GitHub
Security Advisory for private handling.

## Scope

This code applies to project spaces (repository, issues, PRs/MRs) and to
direct contact in a project-related context (email, forum, chat). It does
not govern conduct in unrelated personal spaces.
